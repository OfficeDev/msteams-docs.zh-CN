# <a name="designing-your-microsoft-teams-app-with-ui-templates"></a>使用 UI 模板设计 Microsoft 团队应用程序

使用 UI 模板更快地设计 Microsoft 团队应用程序。 这些模板是一个基于 UI 的熟知组件的集合，这些组件在常见的团队使用案例中工作，为您提供更多的时间来找出您的用户的最佳体验。

## <a name="microsoft-teams-ui-kit"></a>Microsoft 团队 UI 工具包

您可以从 Microsoft 团队 UI 工具包中获取 UI 模板，其中还包括有关使用情况、剖析、辅助功能和最佳做法的广泛信息。

> [!div class="nextstepaction"]
> [ (Figma) 获取 Microsoft 团队 UI 工具包 ](https://www.figma.com/community/file/916836509871353159)

## <a name="list"></a>列表

您可以使用列表以易于浏览的格式显示相关项目，并允许用户对整个列表或单个项目执行操作。

:::image type="content" source="../../assets/images/ui-templates/list.png" alt-text="示例显示一个列表 UI 模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 显示数据
* 对应用程序内容的上下文操作

## <a name="dashboard"></a>仪表板

仪表板在中心位置 (工作组个人应用程序或选项卡) 中显示不同类型的内容。 用户应至少能够自定义其在仪表板上看到的部分内容。

:::image type="content" source="../../assets/images/ui-templates/dashboard.png" alt-text="示例显示仪表板 UI 模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 分析数据
* 报告指标
* 在一个位置组织不同的信息

## <a name="form"></a>表单

表单用于以结构化方式收集、验证和提交用户输入。 清除输入字段的标记和逻辑分组对于出色的用户体验至关重要。

:::image type="content" source="../../assets/images/ui-templates/form.png" alt-text="示例显示一个表单 UI 模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 登录
* 用户配置文件
* 设置
* 用户输入集合

## <a name="sign-in"></a>登录

您可以为不同的团队上下文和标识提供程序设计应用登录流。 下面的示例包含 (SSO) 的单一登录，我们建议采用最简单的身份验证体验。

:::image type="content" source="../../assets/images/ui-templates/sign-in.png" alt-text="示例显示了登录 UI 模板。" border="false":::

### <a name="top-use-case"></a>主要用例

* 对用户进行身份验证

## <a name="task-board"></a>任务板

任务板（有时称为 "看板母板" 或 "泳道航道"）是用于跟踪工作项或票证状态的卡片集合。 它还可用于按类别对任何类型的内容进行排序。 您可以编辑和移动各列之间的卡片。

:::image type="content" source="../../assets/images/ui-templates/task-board.png" alt-text="示例显示了任务板 UI 模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 项目管理。 分配任务和跟踪状态
* 发表. 在不同类别中添加想法
* 对练习进行排序。 将任何类型的信息组织到存储桶中

## <a name="data-visualization"></a>数据可视化

您可以使用不同的卡片大小 (single、double 和 full) 在同一页上堆叠和组织数据可视化。 要适应列布局并填充空格的卡片进行缩放。

:::image type="content" source="../../assets/images/ui-templates/data-viz.png" alt-text="示例显示一个数据可视化用户界面模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 显示复杂信息
* 创建仪表板

## <a name="wizard"></a>向导

向导通过几个屏幕指导用户完成任务 (如安装过程) 。

:::image type="content" source="../../assets/images/ui-templates/wizard.png" alt-text="示例显示一个向导 UI 模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 设置
* 载入
* 首次运行体验

## <a name="empty-state"></a>空状态

空状态模板可用于多种方案，包括登录、首次运行体验、错误消息等。 非常灵活—可将其调整为使用以下设计中的一个、几个或全部组件。

:::image type="content" source="../../assets/images/ui-templates/empty-state.png" alt-text="示例显示一个向导 UI 模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 登录
* 欢迎消息和首次运行体验
* 成功消息
* 错误消息

## <a name="notification-bar"></a>通知栏

通知栏是一个专用区域，用于显示简短的重要邮件，不要求用户立即采取措施。 特定的背景颜色和图标与特定类型的邮件相关联 (请参阅下) 。

:::image type="content" source="../../assets/images/ui-templates/notification-bar.png" alt-text="示例显示通知栏模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 严重邮件、错误和警告
* 成功消息
* 信息或促销消息

## <a name="left-nav"></a>左侧导航

使用左侧导航浏览 "团队" 选项卡中的多个页面。在下面的示例中，左侧导航位于频道列表和选项卡内容之间。

:::image type="content" source="../../assets/images/ui-templates/left-nav.png" alt-text="示例显示了一个左侧导航模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 浏览 "团队" 选项卡中的多个页面
* 将复杂应用拆分为多个页面

## <a name="breadcrumb"></a>痕迹导航栏

痕迹导航是传达应用程序层次结构的导航辅助工具。 它们可帮助用户了解其查看页面的总体体验，并在该层次结构中单击一次单击即可访问更高级别。

:::image type="content" source="../../assets/images/ui-templates/breadcrumb.png" alt-text="示例显示了痕迹导航模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 通信层次结构
* 导航

## <a name="toolbar"></a>工具栏

工具栏是用于对一组控件进行分组的容器。

:::image type="content" source="../../assets/images/ui-templates/toolbar.png" alt-text="示例演示了一个工具栏模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 对应用程序内容的上下文操作
* 上下文筛选器和查找
* 导航和痕迹导航

## <a name="stage"></a>阶段

Stage 提供一种方法，使用户能够在团队中打开实体（如图像、文件或网站），而不是在另一个应用程序或浏览器中打开它。 舞台的主要用例正在查看;曲面不应用于复杂的交互。

 (实现注意：使用大型 [任务模块](../../task-modules-and-cards/task-modules/design-teams-task-modules.md)构建你的阶段。 ) 

:::image type="content" source="../../assets/images/ui-templates/stage.png" alt-text="示例显示了一个暂存模板。" border="false":::

### <a name="top-use-cases"></a>主要用例

* 在团队中打开实体，而不是另一个应用程序或浏览器
* 聚焦媒体或其他内容
